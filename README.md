# Listify — Real Estate Listing SPA

> **Archived / Not Production-Ready** — Development on this project has reached a stable point but has since been superseded by an Angular equivalent. This repository is preserved as a reference and serves as an overview of React technical skills, architecture decisions, and frontend patterns. It is not actively maintained.

## Overview

Listify is a single-page application for real estate listings, supporting property search, posting, messaging, and map-based exploration. It is built with a modern React stack and targets a multilingual audience (EN/RO/RU).

---

## Tech Stack

| Layer | Technology |
|---|---|
| UI Framework | React 19 + TypeScript (strict mode) |
| Build Tool | Vite 7 (dev server on port 4200) |
| Routing | TanStack Router (file-based, auto code splitting) |
| Server State | TanStack Query |
| Client State | Zustand |
| Styling | Tailwind CSS v4 + Shadcn/UI (Radix primitives) |
| Icons | Tabler Icons |
| Forms & Validation | React Hook Form + Zod |
| Real-time | Socket.io-client |
| Maps | Google Maps via `@vis.gl/react-google-maps` |
| Animations | Framer Motion |
| i18n | i18next (EN / RO / RU) |

## Getting Started

### Prerequisites

- Node.js 20+
- A running instance of the main backend on `localhost:8080`

### Install & Run

```bash
npm install
npm run dev        # Start Vite dev server → http://localhost:4200
```

### Available Scripts

```bash
npm run dev              # Start dev server (port 4200)
npm run build            # TypeScript check + production build (tsc -b && vite build)
npm run lint             # Run ESLint
npm run preview          # Preview production build locally
npm run serve-build      # Serve built dist on port 4200
npm run pull-translations  # Sync i18n translations from i18nexus
```

## Key Technical Decisions

A few decisions worth calling out for anyone reviewing the architecture:

- **Custom ESLint rule for domain isolation** (`eslint-rules/no-domain-deep-imports`) — domain boundaries are enforced at lint time, not just by convention. No domain can import another domain's internals; all cross-domain shared code must go through `src/shared/`. This makes refactoring domain internals safe.

- **Compile-time RBAC** — the permission system in `src/domains/auth/hooks/use-permissions.ts` uses TypeScript discriminated unions to map roles → resources → actions. Invalid combinations are a type error, not a runtime bug.

- **Race-condition-safe token refresh** — the Axios interceptor in `src/shared/services/lib/axios.ts` uses a double-checked locking pattern to ensure concurrent 401 responses trigger only one token refresh, with all queued requests replayed after resolution.

- **Socket-first with polling fallback** — React Query's `refetchInterval` is a function that checks socket connection state. When the socket is live it returns `false` (no polling); when disconnected it falls back to a 30-second poll automatically.

- **State layer separation** — Zustand handles client/UI state (theme, language, modals) with localStorage persistence. React Query owns all server state. The two layers do not overlap.

## Architecture Notes

### API Proxies (dev)

Configured in `vite.config.ts`:

| Prefix | Target |
|---|---|
| `/api` | `http://localhost:8080` (main backend) |
| `/api/node` | `http://localhost:8085` (Node backend) |

### Path Alias

`@/*` resolves to `./src/*` in both TypeScript and Vite.

### Routing

Routes are auto-generated in `src/routes/routeTree.gen.ts` by TanStack Router. **Do not edit this file manually.**

### Domain-Driven Import Rules

A custom ESLint rule (`no-domain-deep-imports`) enforces that domains do not import each other's internals. Cross-domain shared code must live in `src/shared/`.

### Theming

Color schemes and dark mode CSS variables are defined in `src/index.scss`. All components must support both light and dark mode.

### Animations

Shared Framer Motion variants (fadeInUp, fadeInDown, scaleIn, stagger containers) are exported from `src/shared/lib/animations.ts`.

### Internationalization

Translation keys live in `src/assets/locales/en/default_en.json`. When adding new UI text, add the key to the English file only — other languages are synced via i18nexus.

### Code Style

Prettier config (`prettier.config.json`): `semi: true`, `singleQuote: true`, `trailingComma: "none"`, `printWidth: 80`, with `prettier-plugin-tailwindcss`.

## Notes

- No test framework is configured. Tests were intentionally deferred — the product was evolving rapidly and writing tests against frequently changing requirements would have meant constant rewrites. The plan was to introduce a test suite once the application stabilized and core flows were locked in.
- The `server/` directory contains a minimal Express server for serving the production build.
