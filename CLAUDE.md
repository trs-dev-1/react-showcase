## Commands

```bash
npm run dev          # Start Vite dev server on port 4200
npm run build        # TypeScript check + Vite build (tsc -b && vite build)
npm run lint         # Run ESLint
npm run preview      # Preview production build
npm run serve-build  # Serve built dist on port 4200
npm run pull-translations  # Sync i18n translations from i18nexus
```

No test framework is configured — there are no tests to run.

## Architecture

**Real estate listing SPA** (Listify) with domain-driven structure.

### Tech Stack
- **React 19** + **TypeScript** (strict mode)
- **Vite 7** (dev server & build; port 4200)
- **TanStack Router** (file-based routing, auto code splitting) + **TanStack Query** (server state)
- **Zustand** (client state)
- **Tailwind CSS v4** + **Shadcn/UI** (Radix + Tailwind, icon library: Tabler)
- **React Hook Form** + **Zod** (forms & validation)
- **Socket.io-client** (real-time messaging)
- **i18next** (EN/RO/RU translations in `src/assets/locales/`)
- **Google Maps** via `@vis.gl/react-google-maps`

### Folder Structure

```
src/
├── domains/         # Feature modules (auth, property, search, messages, admin, etc.)
├── routes/          # TanStack Router file-based routes (mirrors domain structure)
├── shared/          # Cross-domain code
│   ├── components/  # Reusable components including shadcn/UI in ui/
│   ├── fetchers/    # Data fetching utilities
│   ├── hooks/       # Custom React hooks
│   ├── services/    # API service layer (Axios)
│   ├── state/       # Zustand stores
│   ├── schemas/     # Zod validation schemas
│   └── providers/   # React context providers
├── assets/locales/  # i18n translation files
└── environment/     # Environment config
server/              # Express SSR/static server (production)
eslint-rules/        # Custom ESLint rule: no-domain-deep-imports
```

### Domain-Driven Import Rules

A custom ESLint rule (`no-domain-deep-imports`) enforces that domains do not reach into each other's internals. Cross-domain shared code lives in `src/shared/`.

### API Proxies (dev)

Configured in `vite.config.ts`:
- `/api` → `http://localhost:8080` (main backend)
- `/api/node` → `http://localhost:8085` (Node backend)

### Path Alias

`@/*` maps to `./src/*` in both TypeScript and Vite config.

### Routing

Routes are auto-generated in `src/routes/routeTree.gen.ts` by TanStack Router — do not edit this file manually.

### Prettier Config

`prettier.config.json`: `semi: true`, `trailingComma: "none"`, `singleQuote: true`, `printWidth: 80`, with `prettier-plugin-tailwindcss`.

### Development Notes

- Use the color schemes from index.scss. The dark mode variables are defined there too. When modifying or add a style for
a component always think dark/light mode.
- Try to split the components when creating new ones if the codebase generated is too large. Identify commonly used logic and split them across custom hooks or utility functions depending on the case.
- Refer to the UI kit components that's under @src/shared/components/ui. Do not build UI primitives from scratch.
Refer to them and only adjust those primitives in the context where they are used if needed.
- The application supports multiple languages. Whenever you have to add a label, use the i18n package and add the key if missing inside default_en.json from @src/assets/locales/en/default_en.json. Do not add the key for the other languages, only form English.
- The animations using framer-motions are created and exported from @src/shared/lib/animations.ts
